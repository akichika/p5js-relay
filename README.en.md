# ✳ p5.js Relay

English | [日本語](README.md)

One-tap transfer of code from Claude, ChatGPT, or Gemini into the p5.js Web Editor and more. Great for vibe coding.

A Chrome / Microsoft Edge (Chromium-based browsers) extension (Manifest V3). A single generated HTML file is **automatically split into index.html / style.css / sketch.js** and pasted in.

- Repository: https://github.com/akichika/p5js-relay
- Bug reports and pull requests are welcome on the repository above

## Installation (Developer Mode)

### Chrome
1. Place this folder anywhere on your machine
2. Open `chrome://extensions` in Chrome
3. Turn on "Developer mode" (top right)
4. Click "Load unpacked" and select this folder

### Microsoft Edge
1. Place this folder anywhere on your machine
2. Open `edge://extensions` in Edge
3. Turn on "Developer mode" (bottom left)
4. Click "Load unpacked" and select this folder

(Edge is Chromium-based, so it can load Chrome extensions as-is. The `chrome.*` APIs work the same way in Edge.)

## Usage

1. A "✳ Transfer" button appears on code blocks / the Canvas panel / the Artifact panel in AI chat
2. Click it to apply the code to your default destination in one tap (a tab is opened automatically if none exists)
3. Use "▾" to pick a destination, or send selected text via the right-click menu

## File Splitting (Multi-file Transfer)

A single HTML file generated by Canvas and similar features is parsed and split to match the format of the destination editor.

**Common splitting logic:** inline `<style>` becomes CSS, inline `<script>` becomes JS, and external `<script src>` (CDN libraries) or `<link rel="stylesheet">` (fonts, etc.) are extracted as "libraries." If the JS is a p5 sketch (has `setup`/`draw`) but doesn't load p5.js, the CDN tag is automatically added.

**Per-destination behavior (splitMode):**

| Mode | Target | Behavior |
|---|---|---|
| `tabs` + merge | p5.js Web Editor | Applied to the style.css / sketch.js tabs. **index.html is not replaced** — its existing content is read first, and only the missing library tags (CDNs or fonts beyond p5's defaults) are inserted just before `</head>`. The editor's built-in p5.js loading is left untouched |
| `tabs` + merge | OpenProcessing | When transferring multiple files, the SKETCH panel's **mode is automatically switched to "HTML/CSS/JS,"** and the code is applied to the mySketch.js / style.css / index.html tabs. index.html uses the same library-merge approach as p5.js (falls back to a full replace on failure). The mode isn't changed for JS-only transfers |
| `panels` | CodePen | The HTML panel receives library `<script>` tags plus body content, the CSS panel receives CSS, and the JS panel receives JS, applied via panel selectors (`#box-*`) |
| `off` | Any | Applied as-is, with no splitting |

- Pure JS-only or CSS-only input is applied directly to the corresponding file
- Limitation: if the generated HTML's body contains required DOM elements, the merge approach may require adding them to index.html manually (since only libraries are auto-inserted; the full-replace fallback used when merging fails does include the body)
- Support for JSFiddle was dropped because pasting there was unreliable (v2.1.3)

## Settings

- **Theme**: System (default) / Light / Dark, selectable on the options page
- **Language**: Automatically follows your system language. Supports English, Japanese, Chinese (Simplified/Traditional), Korean, Spanish, French, German, Portuguese (Brazil), and Russian — 10 languages total (unsupported languages fall back to English)
- **Editor type**: `auto` (recommended — automatically detects CM5 → CM6 → Monaco → Ace → textarea → contenteditable, in that order), or a fixed type. Even with a fixed type, it falls back to auto if nothing is found within 5 seconds
- **Paste mode**: Replace (default) / Append
- **About**: Links to the GitHub repository (Issues/Pull Requests) and X account are shown at the bottom of the options page

## Supported Sites

- Sources: claude.ai / chatgpt.com / chat.openai.com / gemini.google.com (Canvas supported) / aistudio.google.com
- Default destinations: p5.js Web Editor / OpenProcessing / CodePen (you can freely add more in the options page)

## How It Works

- A content script on the source site watches `pre` elements and Canvas panels (including Shadow DOM) and attaches the button
- The service worker splits a single HTML file using regular expressions (since service workers have no DOMParser)
- `chrome.scripting.executeScript` runs in the **MAIN world** to directly call the CodeMirror / Monaco / Ace API on the destination page. For multi-file transfers, file tabs are switched via click events and applied one by one
- Extraction from Canvas panels also uses the MAIN world to read the full content from the editor API (to handle CM6's virtual scrolling)
